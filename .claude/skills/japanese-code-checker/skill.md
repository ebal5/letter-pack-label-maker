# Japanese Code Checker Skill

このSkillは、日本語コード（コメント、ドキュメント、文字列リテラル）の品質をチェックします。

## 使用方法

ユーザーが以下のような依頼をした際に、このSkillを起動してください：
- 「日本語のコメントをチェックして」
- 「ドキュメントの文体を統一して」
- 「文字コーディングを確認して」
- 「全角・半角が混在していないか確認して」

## チェック項目

### 1. 文字コーディング

すべてのテキストファイルがUTF-8かチェック：
- Pythonファイル（`.py`）
- Markdownファイル（`.md`）
- HTMLファイル（`.html`）
- CSVファイル（`.csv`）
- 設定ファイル（`.yaml`, `.toml`, `.json`）

**検出される問題：**
- Shift_JIS、EUC-JPなどのレガシーエンコーディング
- BOM付きUTF-8（不要な場合）
- 混在するエンコーディング

### 2. 全角・半角の統一

日本語文中での使い分けをチェック：

**推奨ルール：**
- **数字**：半角（例：`123` ✅、`１２３` ❌）
- **英字**：半角（例：`ABC` ✅、`ＡＢＣ` ❌）
- **記号（一部）**：半角（例：`()` ✅、`（）` ❌）
- **日本語の句読点**：全角（例：`、。` ✅、`,.` ❌）
- **括弧（日本語文中）**：全角（例：`（）` ✅、`()` ❌）

**例外：**
- コード例内は半角を使用
- URLは半角のみ
- Markdown見出しの記号は半角

### 3. 文体の統一

ドキュメントごとに文体を統一：

**README.md, AGENTS.md, CLAUDE.md：**
- 「です・ます」調（丁寧語）
- 一貫した敬語レベル

**コメント：**
- 簡潔な説明文（体言止めも可）
- 統一された文末（。で終わるか、終わらないか）

**エラーメッセージ：**
- ユーザー向け：丁寧語
- 開発者向け：簡潔な説明

### 4. コメントの充実度

重要な関数・クラスに適切なコメントがあるかチェック：

**必須コメント：**
- クラスのdocstring
- 公開関数のdocstring
- 複雑なロジックの説明コメント

**推奨コメント：**
- 引数の説明
- 戻り値の説明
- 使用例

### 5. 用語の統一

プロジェクト全体で用語を統一：

**例：**
- 「ラベル」vs「レーベル」
- 「郵便番号」vs「〒番号」
- 「住所」vs「アドレス」
- 「氏名」vs「名前」

## 実装の詳細

このSkillは以下のツール/スクリプトを使用します：

- `tools/check_japanese_code.py` - メインのチェックスクリプト

### 出力例

```
🔍 日本語コードチェックレポート

【文字コーディング】
✅ すべてのファイルがUTF-8です

【全角・半角の使い分け】
⚠️ 3件の問題を検出

src/letterpack/label.py:45
  - 問題: 全角数字が使用されています
  - 現在: 「郵便番号は７桁です」
  - 推奨: 「郵便番号は7桁です」
```

## Skillの起動フロー

1. ユーザーが日本語品質チェックに関する依頼をする
2. このSkillが起動され、check_japanese_code.pyを実行
3. チェック結果をレポート形式で出力
4. 必要に応じて修正案を提示
